# Q4. Evaluating Text Search

**Answer:** **0.76**

Measured values:

- Hit Rate = **0.7583**
- MRR = **0.5943**

## Explanation

Hit Rate measures whether the correct page appears anywhere within the top five retrieved results.

This evaluation focuses purely on retrieval quality before introducing any language model generation, allowing the search component to be evaluated independently.

## Code

```python
def compute_relevance(q, search_function):
    filename = q["filename"]
    results = search_function(q["question"])

    return [
        int(d["filename"] == filename)
        for d in results
    ]


def compute_relevance_total(ground_truth, search_function):
    return [
        compute_relevance(q, search_function)
        for q in tqdm(ground_truth)
    ]


def hit_rate(relevance):
    return sum(
        1 for line in relevance
        if 1 in line
    ) / len(relevance)


def mrr(relevance):
    total = 0.0

    for line in relevance:
        for rank, val in enumerate(line):
            if val == 1:
                total += 1 / (rank + 1)
                break

    return total / len(relevance)


def evaluate(ground_truth, search_function):
    relevance_total = compute_relevance_total(
        ground_truth,
        search_function
    )

    return {
        "hit_rate": hit_rate(relevance_total),
        "mrr": mrr(relevance_total)
    }


result_text = evaluate(
    ground_truth,
    text_search
)

result_text
```

### Output

```python
{
    'hit_rate': 0.7583333333333333,
    'mrr': 0.5942592592592594
}
```

## Key Takeaway

Compared with the course implementation, only the relevance condition changed—from matching document IDs to matching filenames.

The evaluation functions (`hit_rate`, `mrr`, and `evaluate`) remain unchanged, demonstrating the benefit of writing generic evaluation code that accepts any retrieval function.

# Q6. Tuning Hybrid Search

**Answer:** **k = 1**

(No tie occurred; this value produced the highest score.)

## Explanation

Hybrid search combines keyword and vector rankings using Reciprocal Rank Fusion (RRF).

The parameter `k` determines how strongly rank positions influence the final score.

- Smaller values place greater emphasis on the highest-ranked documents.
- Larger values reduce the difference between nearby ranks, giving more weight to documents that appear consistently across both rankings.

Although the original RRF paper recommends `k = 60`, the optimal value depends on the dataset being evaluated.

## Code

```python
results_by_k = {}

for k in [1, 50, 100, 200]:

    result = evaluate(
        ground_truth,
        lambda query, k=k: hybrid_search(query, k=k)
    )

    results_by_k[k] = result

    print(f"k={k}: {result}")

best_k = max(
    results_by_k,
    key=lambda k: (
        results_by_k[k]["mrr"],
        -k
    )
)

best_k
```

Only `k = 1` yielded a measurable improvement, indicating that giving greater importance to each retrieval method's top-ranked results produced the strongest hybrid ranking.

# Summary

| Search Method         |   Hit Rate |        MRR |
| --------------------- | ---------: | ---------: |
| Text Search           | **0.7583** | **0.5943** |
| Vector Search         | **0.7250** | **0.5486** |
| Hybrid Search (k = 1) | **0.8389** | **0.6482** |

Hybrid search achieved the highest performance across both evaluation metrics by combining the strengths of keyword and vector retrieval through Reciprocal Rank Fusion.

One particularly noteworthy observation is that keyword search outperformed vector search on this dataset. While this may appear counterintuitive, it reflects the characteristics of the generated evaluation questions, which retained vocabulary from the original lesson pages.

Overall, the homework demonstrates an important principle of information retrieval: decisions should be driven by measured evaluation metrics rather than assumptions about which retrieval technique is expected to perform better.
